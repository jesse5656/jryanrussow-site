# ChatGPT Project Workspace Standard

Version: 1.0.0

Status:
Active Standard

## Purpose

Define the standard workspace structure for The Russow Institute project in ChatGPT.

The objective is to reduce cognitive switching costs, keep project boundaries clear, and prevent unrelated work from bleeding across chats.

---

## Project

The parent project is:

The Russow Institute

The project contains three primary long-running chats.

## Boundary Rule

Each chat should remain focused on its primary domain.

Research belongs in #1.

Product engineering belongs in #2.

Business execution belongs in #3.

When a topic crosses domains, clearly label it as one of the following:

- Field Observation
- Engineering Request
- Operational Validation Request
- Feature Request
- Design Constraint

---

## Workflow Principle

The flow is:

Research

↓

Engineering

↓

Operations

Then feedback flows upward:

Operations

↓

Engineering

↓

Research

This creates a continuous improvement loop.

---

## Core Rule

Do not let one chat become the dumping ground for all work.

The purpose of the workspace is separation of concerns.
