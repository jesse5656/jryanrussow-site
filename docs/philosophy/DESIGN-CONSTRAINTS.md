# Design Constraints

Version: 0.1.0

Every framework developed by the Russow Institute should satisfy these constraints.

------------------------------------------------------------------------------

It should become easier to understand over time.

It should become easier to teach over time.

It should become easier to implement over time.

It should become easier to improve over time.

It should become easier to transfer over time.

------------------------------------------------------------------------------

Frameworks should reduce unnecessary complexity.

------------------------------------------------------------------------------

Frameworks should not require extraordinary people to function.

------------------------------------------------------------------------------

Frameworks should remain useful even as technology changes.

------------------------------------------------------------------------------

Frameworks should survive leadership transition.

------------------------------------------------------------------------------

Frameworks should produce better stewards.

------------------------------------------------------------------------------

If a framework violates these constraints,
it should be redesigned.
