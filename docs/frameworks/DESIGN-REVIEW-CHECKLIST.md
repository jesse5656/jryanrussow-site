# Design Review Checklist

A framework is not Canon until it satisfies this review.

------------------------------------------------------------------------------

Definition

□ Clearly defined

------------------------------------------------------------------------------

Purpose

□ Solves a meaningful problem

------------------------------------------------------------------------------

Research

□ Evidence collected

------------------------------------------------------------------------------

Case Studies

□ Tested in practice

------------------------------------------------------------------------------

Competing Explanations

□ Considered

------------------------------------------------------------------------------

Implementation

□ Repeatable

------------------------------------------------------------------------------

Transferability

□ Teachable

------------------------------------------------------------------------------

Stewardship

□ Creates better stewards

------------------------------------------------------------------------------

Revision

□ Open questions documented
